📱 AppStudio Mobileとは？

スマートフォンやタブレットのブラウザ上で動作する、高機能なコードエディタ＆実行環境です。
「ChatGPTやClaudeなどのAI」と連携することを前提に設計されており、 プロンプト作成からコード修正までをスムーズに行うための専用ツールを搭載しています。
CodeMirror等のエディタ部品はCDNから読み込みますが、アプリのコードやローカル保存データを自動で外部送信することはありません。

🚀 開発のワークフロー

  • 1
    AI指示でプロンプトをコピー
    「🤖 AI指示」から作りたい内容を書き、AIに渡すプロンプトをコピーします。
  • 2
    AIの回答を全コピー
    外部AIチャットで生成されたHTMLフルコード、またはJSONパッチの回答をそのままコピーします。
  • 3
    「🔁 反映」ワンタップで自動判定
    実行画面の「🔁 反映」を押すと、クリップボード内容をHTMLフルコードかJSONパッチか自動判定して反映します。
  • 4
    修正はJSONパッチで同じ流れ
    追加修正はJSONパッチをAIに出してもらい、同じようにコピーして「🔁 反映」します。

💾 保存とプロジェクト管理

「💾 保存」メニューから、以下の形式で出力できます。

  • アプリ保存（プロジェクト機能）: ブラウザ内のローカルDBに保存します。
    アプリ名を入力するとそれが「プロジェクト名」となり、保存のたびに日時が付与され自動的にバージョン管理が行われます。
    ライブラリ画面ではプロジェクト単位で階層表示され、過去のバージョンへ簡単にアクセスできます。
  • HTML DL: 単一のHTMLファイルとしてダウンロード。Webサーバーにアップすればそのまま公開可能。
  • GitHub Repo: 単一HTMLまたはPWAアプリとして直接公開できます。PWAを選んでも編集・ライブラリ保存の正本は単一HTMLのままで、公開時だけindex.html、manifest、Service Worker、アイコンをメモリ上で一時生成し、指定フォルダへ1コミットで反映します。

🔒 セキュリティと制約

・外部通信: CodeMirror等のエディタ部品はCDNから読み込みます。作成アプリからの外部通信は制限していないため、信頼できないコードは貼らないでください。
・データ保存: アプリのコードやローカル保存データを自動で外部送信することはありません。データはブラウザ内に保存されますが、キャッシュクリアすると消える可能性があるため、重要なコードはGitHub等へバックアップしてください。
・互換実行（初期状態）: 作成アプリはAppStudio本体と同じlocalStorage・IndexedDB空間を使用します。作成アプリがlocalStorage.clear()を実行すると、本体のGitHubリポジトリ設定や開発統計が消える可能性があります。作成アプリがindexedDB.deleteDatabase("AppStudioDB")を実行すると、ライブラリや保存アプリが消える可能性があります。複数の作成アプリが同じlocalStorageキーを使用すると、セーブデータが衝突する可能性があります。また、作成アプリから親画面のDOMや関数へアクセスできる可能性があります。信頼できないコードを実行しないでください。
・安全実行: 実行画面の「その他」メニューから切り替えます。作成アプリからAppStudio本体へのアクセスを制限しますが、localStorageやIndexedDBが使用できない場合があり、通常実行のセーブデータが共有されない場合があります。その際SecurityErrorが発生する場合があります。未確認コードの最初の動作確認に向いていますが、完全なセキュリティ保証ではありません。
・別窓実行: window.openerによる親画面の直接参照を遮断します。ただしAppStudio本体と同一オリジンで動作し、localStorage・IndexedDBは本体と共有します。iframeの安全実行とは異なるため、信頼できるコードだけを実行してください。
・GitHubトークン: 保存せず、この画面を開いている間だけメモリ上に保持します。
